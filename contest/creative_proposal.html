01 / PROBLEM

销售痛点

销售每天要跟进几十个客户，凭经验判断、凭感觉跟进，缺乏数据支撑

🎲

经验依赖

依赖人工经验判断客户意向，新手销售难以快速成长，决策准确率低

📊

数据分散

客户信息分散在微信、企微、邮件等多个渠道，无法统一分析和管理

⏰

时机错失

跟进时机凭感觉，错失最佳购买意向期，导致客户流失

04 / FORMULA

决策公式体系

五大销售决策公式，科学量化客户状态，智能判断跟进策略

BQ · 购买意愿真实度

穿透客户反话和假性拒绝，准确识别真实购买意向

BQ = Sp × 0.3 + Fback × 0.2
+ Investment × 0.3 + (1-Pface) × 0.2

# Sp: 客户主动联系比例
# Fback: 客户回复字数比
# Investment: 客户投入的时间/参与度
# Pface: 客户隐藏真实预算的程度

BSP · 商务势能

监控权力平衡，判断销售与客户的互动地位

BSP = f(user_ddepth, target_ddepth,
target_latency, user_latency)

# user_ddepth: 销售的急迫程度
# target_ddepth: 客户的紧迫程度
# target_latency: 客户的回复速度
# user_latency: 销售的回复速度

BWS · 购买意向期

决定何时出手，把握最佳跟进时机

BWS = f(gap_effect, cp_index,
eev, scarcity_loss)

# gap_effect: 差距效应
# cp_index: 竞争压力指数
# eev: 情绪期望值变化
# scarcity_loss: 稀缺性损失感知

PV · 成交期望值

评估成交概率与潜在收益，优化资源分配

PV = p_succ × escalation_bonus
- p_fail × loss_risk

# p_succ: 成功概率
# escalation_bonus: 成交收益
# p_fail: 失败概率
# loss_risk: 失败风险

sales_action · 终极决策

def sales_action(bq, bsp, bws, bs=0.0, pv=0.5):
if bq > 1.0 and bws > 0.8 and pv > 0.3:
return "报价/逼单" # bargain
elif bq > 0.7 and bsp >= 0.6:
return "推进" # push
elif bs > 0 and bws > 0.5:
return "培育" # nurture
elif bsp < 0.6:
return "重置关系" # reset
else:
return "维持" # maintain
