## 1. 创意名称 + 创意介绍

**创意名称**：SalesCRM — AI 销售分析助手

**想解决什么问题**：销售每天要跟进几十个客户，凭经验判断谁是高意向客户、凭感觉决定跟进时机，缺乏数据支撑导致效率低下、错失良机。

**为什么会想到做这个**：观察到销售日常跟进缺乏量化决策工具，灵感来自对客户行为模式的系统化建模。通过「代码负责数据，Agent 负责推理」的核心原则，让数据驱动销售决策。

**大概是什么产品**：本地优先的 AI 销售分析助手。底层用 Python 工具同步和量化微信客户数据，上层通过 Agent 工作流生成客户判断、跟进优先级和可直接发送的话术。

---

## 2. 目标用户及痛点

**面向哪些用户**：

- B2B 销售代表
- B2C 客户经理
- 销售团队管理者
- 客户成功经理

**在什么场景下使用**：

- 日常客户跟进分析
- 周报/月报数据汇总
- 商机评估与优先级排序
- 客户维护与流失预警

**当前痛点**：

- 🎲 **经验依赖**：依赖人工经验判断客户意向，新手销售难以快速成长，决策准确率低
- 📊 **数据分散**：客户信息分散在微信、企微、邮件等多个渠道，无法统一分析和管理
- ⏰ **时机错失**：跟进时机凭感觉，错失最佳购买窗口期，导致客户流失
- 📋 **缺乏系统化工具**：没有统一的客户分析和决策辅助平台

---

## 3. 价值与意义

**效率提升**：

- 自动化分析客户状态，减少人工判断时间，让销售专注于高价值的客户沟通和成交动作
- 16 个加权量化指标 + 5 个销售决策公式，科学评估客户状态，将主观经验转化为客观数据

**商业价值**：

- 通过 BQ（购买意愿真实度）、BSP（商务势能）、BWS（购买窗口期）、PV（成交期望值）等公式，精准识别高意向客户
- 智能推荐跟进优先级，优化资源分配，提升成交率和客户转化率
- 降低决策风险，提升团队整体作战能力

---

## 4. 核心功能亮点

- 🔄 **多渠道数据同步**：从微信、企微、CRM、邮件等多渠道实时同步客户沟通数据
- 📈 **智能指标引擎**：自动计算反馈率、响应延迟、质量评分等 16 个加权指标
- 🧠 **AI 决策分析**：Agent 调用工具获取数据，结合销售知识库和决策公式提供个性化建议
- 👤 **身份目录管理**：Person→Account→Alias 三层映射，支持多账号统一管理
- 🔍 **事件自动检测**：自动识别需求确认、决策者出现、方案发送等关键销售事件
- 📋 **客户排名视图**：热门/沉默/紧迫列表，智能推荐跟进优先级

---

## 5. 技术架构

```
用户 → Web UI → Python 工具层 → SQLite 数据层
                    ↓
            知识库 + 决策公式
```

**核心原则**：代码负责数据，Agent 负责推理

**技术栈**：Python 3.13、SQLite、销售指标引擎、Agent 工具层、微信数据同步；后续可扩展 FastAPI/Vue 前端、企微和 CRM 接入

---

## 6. 发展路线

### 当前阶段：Agent 工作流编排

将数据读取、指标计算、知识库检索、决策建议封装成稳定流程，输出结构化建议：当前阶段、证据、风险、下一句话怎么发。

### 未来扩展

- Demo 数据模式：支持样例 SQLite 数据跑通，无需真实微信环境
- Web UI 层：客户排名、客户分析、跟进建议三个核心页面
- MCP 工具服务：后续可扩展为标准工具服务，供外部 AI Agent 调用

## 7. Demo 体验闭环

1. **左侧**：客户列表 — 热客、沉默客户、紧急客户排名
2. **中间**：客户详情 — 最近聊天、关键事件、指标雷达/分数
3. **右侧**：AI 分析面板 — 用户输入"张三能不能报价？""今天该跟进谁？"
4. **后端调用**：sync_person → brief → chat → metrics → sales_params → sales_action
5. **输出**：结构化建议 — 当前阶段、证据、风险、下一句话怎么发
