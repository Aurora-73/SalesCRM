# 客户维护工作流

> 每周周报后执行，识别需要主动联系的客户，并给出具体跟进建议。

---

## 工作流概览

```
rank() → 筛选客户 → metrics()/chat()/evidence() → 分析 → 输出：谁需要跟进 + 怎么跟进
```

---

## 第一步：筛选客户

从排名中筛选需要主动联系的客户，规则：

### 信号 1：客户热度下降（最优先）

| 条件 | 含义 |
|------|------|
| `recent > 3` | 最后消息超过 3 天前 |
| `signal_level` 在"冷淡"及以上 | 有一定基础的客户关系 |
| `trend < -0.005` | 周变化下降 |
| 未被排除（非"系统号"/"企业号"/"放弃"） | 不是无关的联系人 |

### 信号 2：高意向但未推进

| 条件 | 含义 |
|------|------|
| `signal_level` 在"弱意向"或以上 | 有意向但没推进 |
| `recent > 1` | 最近 1 天没联系 |
| 无会面/报价记录（events 无 MEETING/PROPOSAL_SENT） | 还没约过或没推进报价 |

### 信号 3：高潜力但互动不足

| 条件 | 含义 |
|------|------|
| `qscore_personal > 0.5` | 客户主动追问产品细节 |
| `neediness_penalty > 0.9` | 互动正常（不是销售过度主动） |
| `recent > 2` | 2 天没联系 |

### 客户数量控制

每次最多输出 **3-5 人**，按优先级排序：
1. 客户热度下降（信号 1）：优先级最高
2. 高意向未推进（信号 2）：次之
3. 高潜力互动不足（信号 3）：再次

---

## 第二步：获取上下文

对每个客户，Agent 调用以下工具获取分析依据：

```python
brief("客户名", compact=True)        # 全局视图
chat("客户名", recent=30)            # 最近 30 条聊天
metrics("客户名")                    # 指标数据
evidence("客户名", section="timeline") # 客户时间线
sales_params("客户名")               # 量化参数
```

---

## 第三步：分析（Agent 自行推理）

Agent 根据数据判断：

1. **当前销售阶段**：潜客/初步接触/需求确认/方案展示/谈判报价/成交/冷淡停滞...
2. **客户的信号强度**：BQ/BWS 是否明确？是冷淡还是意向？
3. **互动状态**：是否过度跟进？是否需要调整策略？
4. **上次沟通的内容**：是自然结束还是有未解答的问题？客户最后一条消息是什么？
5. **客户最近的动态**：朋友圈有什么？是否有新变化？

---

## 第四步：输出建议

每位客户输出一个**具体的跟进建议**，要求：

### 跟进规则

1. **context-consistent**：必须基于实际聊天内容，不编造不存在的叙事
2. **有价值点**：消息要能让客户有理由回复（分享价值/解决问题/邀请沟通）
3. **匹配阶段**：
   - 潜客获取：建立初步连接，了解需求
   - 初步接触：展示产品价值，建立信任
   - 需求确认：深入了解痛点，确认决策链
   - 方案展示：定制方案，逐步确认意向
   - 谈判/报价：价格谈判，异议处理
   - 冷淡停滞：低频率试探，提供价值而非打扰
4. **简短清晰**：不超过 2 句话，不给客户压力

### 好的跟进示例

- "在吗" → "在吗"（不是好的，太泛）
- "在吗" → "上次你提到的XX问题，我们刚出了解决方案，想和你简单聊下" → **有具体价值 + 解决问题**
- "有空吗" → "本周三下午我在你公司附近，方便约个 15 分钟的产品演示吗" → **具体会面 + 低压力**

### 输出格式

```
## 客户维护提醒（本周）

### 1. 张三（排名 #3，信号：冷淡，最后联系：3 天前）
- 客户状态：需求确认阶段，BQ 0.8，但 3 天没联系
- 上次沟通：客户问了预算问题，你说稍后给方案
- 建议跟进："上次你关心的预算方案我整理好了，发你看下？预计 5 分钟能讲清楚"
- 原理：延续上次话题，提供具体价值，明确时间承诺

### 2. 李四（排名 #5，信号：弱意向，最后联系：5 天前）
...
```

## 工作流触发

两种方式：

**方式 A：每周自动**（建议周五下午）
- 周报完成后自动触发
- 输出 `data/outputs/reports/maintain_weekXX.md`

**方式 B：手动触发**
- 用户说"谁需要跟进"或"客户维护分析"
- 立即执行

---

## 注意事项

1. **不要过度打扰**：如果客户明确冷淡（意向等级=冷淡且 recent > 7），不要建议主动联系，等客户先动或提供高价值内容
2. **不要群发式**：每位客户的消息必须不同，不能套模板
3. **结合当前状态**：如果客户最近朋友圈发了竞品相关内容，建议里要提到这个信号，不要盲目跟进
4. **尊重"放弃"标签**：已标记"放弃"的客户不进入候选
5. **不要超过 5 人**：专注维护高质量客户关系

## 实现方案

写一个 `engine/tools.py` 的工具函数 `maintain_candidates()`：

```python
def maintain_candidates(max_people: int = 5) -> dict:
    """返回需要维护的客户列表。
    
    返回：
    {
        "candidates": [
            {
                "name": "张三",
                "rank": 3,
                "signal_level": "冷淡",
                "recent_days": 3,
                "trend": -0.005,
                "reason": "3天没联系，信号弱意向，BQ中等",
            },
            ...
        ],
        "total_checked": 50,
        "excluded": 45,
    }
    """
```

Agent 拿到候选名单后，再逐个调用 `brief`/`chat`/`metrics` 拿详细数据，自行分析并输出跟进建议。
