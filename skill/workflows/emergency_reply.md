# 紧急回复流程

## 流程概览

```
0: 同步 → 1: 聊天记录 → 2: 指标 → 3: Wiki搜索 → 给出回复建议
```

---

## 第0步：同步最新消息

**工具**：`person_sync(name)` 【MCP工具】

**目的**：确保看到最新消息。

---

## 第1步：查看最近聊天

**工具**：`person_chat(name, recent=30)` 【MCP工具】

**目的**：看最近30条消息理解上下文。

**执行细节**：
- 重点关注：对方最后一条消息、最近互动模式

---

## 第2步：获取指标

**工具**：`person_metrics(name)` 【MCP工具】

**目的**：了解当前客户状态。

**执行细节**：
- 重点关注：signal_level、interaction_pattern

---

## 第3步：搜索话术策略

**工具**：`wiki_search(keyword)` 【MCP工具】

**目的**：搜索需求刺激/价格异议/竞品应对等策略。

**搜索关键词参考**：
| 场景 | 推荐搜索词 |
|------|-----------|
| 对方问"在吗" | 开场技巧 冷读 |
| 对方说"忙" | 频率法则 需求感 |
| 对方说"价格贵" | 价格异议 应对 |
| 对方说"考虑一下" | 逼单时机 窗口期 |
| 对方提及竞品 | 竞品应对 差异化 |

---

## 回复构造规则

1. **一条可直接发送的回复**（主回复）
2. **备选回复**（2-3条，不同风格）
3. **不要说**（禁忌话术）
4. **观察点**（发送后观察什么信号）

---

## 回复模板

```markdown
### 推荐回复
"[可直接发送的回复内容]"

### 备选方案
1. "[备选1]"
2. "[备选2]"

### 不要说
- "[禁忌1]"
- "[禁忌2]"

### 观察点
- 发送后观察回复速度和回复质量
- 如果XX情况出现，下一步XX
```
